### Дорожная карта изучения Golang (4 месяца, интенсив)

**Принцип:** теория → практика → проект. После каждого блока — мини‑проект на GitHub.

---

#### Месяц 1: Базы и синтаксис

**Что учить:**
- Установка Go, настройка GOPATH/GOMODULE.
- Структура программы: `package`, `import`, `func main()`.
- Типы данных: `int`, `float64`, `string`, `bool`, `rune`, `byte`.
- Переменные и константы: `var`, `const`, короткое объявление `:=`.
- Управляющие конструкции: `if`, `else`, `switch`, циклы `for`.
- Массивы, слайсы (`[]int`), карты (`map[string]int`).
- Структуры (`struct`), методы, указатели.
- Функции: аргументы, возвращаемые значения, множественные возвращаемые значения.

**Практика:**
- Калькулятор с CLI.
- Парсер CSV‑файла (чтение/запись).
- Мини‑блог на консольном вводе/выводе.

**Ресурсы:**
- Официальная документация (golang.org/doc/).
- Tour of Go (tour.golang.org).

---

#### Месяц 2: Основы работы с кодом и параллелизм

**Что учить:**
- Пакеты и модули: `go mod init`, `go get`, импорт локальных пакетов.
- Обработка ошибок: `error`, `if err != nil`, `errors.New()`, `fmt.Errorf()`.
- Интерфейсы: объявление, реализация, пустой интерфейс `interface{}`.
- Goroutines: запуск через `go func()`, синхронизация.
- Каналы: буферизованные/небуферизованные, `select`, закрытие каналов.
- Синхронизация: `sync.Mutex`, `sync.WaitGroup`.
- Работа с файлами: `os.Open()`, `ioutil.ReadFile()`, запись.
- JSON: `json.Marshal()`, `json.Unmarshal()`.

**Практика:**
- Многопоточный сканер портов (goroutines + каналы).
- API‑клиент с кэшированием в файл.
- Чат в терминале (клиент‑сервер на каналах).

**Ресурсы:**
- «The Way to Go» (онлайн‑книга).
- GitHub‑репозитории с примерами (искать по метке `golang-examples`).

---

#### Месяц 3: Веб и базы данных

**Что учить:**
- HTTP‑сервер: `net/http`, роутинг, middleware.
- Обработчики: `http.HandlerFunc`, `ServeMux`.
- Шаблоны: `html/template`.
- БД: `database/sql`, драйверы (`pq` для PostgreSQL, `mysql`).
- ORM (опционально): `gorm.io/gorm`.
- REST API: проектирование эндпоинтов, статусы HTTP.
- Аутентификация: JWT, хэширование паролей (`bcrypt`).
- Тестирование: `testing`, `testify/require`.

**Практика:**
- Блог с CRUD‑операциями (постgreSQL/SQLite).
- URL‑шортер с API.
- Сервис прогноза погоды (API + кэш в БД).

**Ресурсы:**
- Go Web Examples (github.com/golang/go/wiki/WebServer).
- Документация `gorm.io`.

---

#### Месяц 4: Продвинутое и продакшен‑навыки

**Что учить:**
- Контекст: `context.Background()`, `context.WithCancel()`, тайм‑ауты.
- Логирование: `log/slog` (стандартный), `uber-go/zap` (производительный).
- Профилирование: `pprof`, анализ CPU/памяти.
- Тестирование: моки (`testify/mock`), бенчмаркинг.
- Деплой: Docker, Kubernetes (базово), CI/CD (GitHub Actions).
- Микросервисы: gRPC (`google.golang.org/grpc`), Protobuf.
- Безопасность: SQL‑инъекции, XSS, CORS.
- Паттерны: синглтон, фабричный метод, worker pool.

**Практика:**
- Микросервис авторизации (gRPC + JWT).
- Система мониторинга с дашбордом (HTTP + Grafana/Prometheus).
- CI/CD для своего проекта (автотесты + деплой в Docker).

**Ресурсы:**
- «Concurrency in Go» (книга).
- Официальные гайды Kubernetes.

---

### Итоговый чек‑лист (через 4 месяца)

Вы должны уметь:
1. Писать чистый код с соблюдением Go‑стилистики (gofmt, golint).
2. Проектировать REST API и gRPC‑сервисы.
3. Работать с SQL/NoSQL БД.
4. Использовать параллелизм без гонок данных.
5. Тестировать код (юнит, интеграционные тесты).
6. Деплоить приложения в Docker.
7. Настраивать логирование и мониторинг.

**Следующий шаг:**
- Углубление в специфичные домены (облака, блокчейн, DevOps).
- Участие в open‑source (поиск проектов по метке `good-first-issue`).
- Подготовка к собеседованиям (LeetCode‑задачи на Go, системные дизайн).
